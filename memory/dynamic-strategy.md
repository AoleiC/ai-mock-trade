# 动态交易策略

本文件是总纲与心法的**具象化操作细则**，供 agent 执行盘面分析与复盘。

> - **流程控制 + 必调接口 + 关键字段含义 + 通用字段消费纪律 + 通用术语表** → 见总纲 `AGENTS.md`
> - **纯交易纪律与风险偏好（五类股、九条铁律、仓位红线、套利边界）** → 见心法 `memory/trading-mindset.md`
> - 本文件只写**本战法特有的字段消费细则 + 选股 / 买点 / 止损 / 风控 / checklist / 输出 / 教训**

## 零、本策略特有的字段消费

> 下列是本战法特有的字段消费细则。通用字段含义见总纲 §3.2，通用字段消费纪律（不自创标签 / 不自拾指标 / 名单非门槛 / 信号≠动作 / 持仓必采分时）见总纲 §5.1，字段缺位判定见总纲 §5.2，退化规则见总纲 §5.3 / §5.4，不以偏概全见总纲 §5.5。

### 0.1 仓位采用与开仓触发

**仓位（直接采用，不自行调档）**

- **唯一字段**：`final_analysis.position_limit`
- **直接采用**作为本轮**新增可动用**仓位上限，**不主动降、不主动加**；`position_limit` 只约束"是否还能开新仓"，**已有持仓不因 `position_limit` 降低而被强卖**（持仓去留归 §三 止盈止损 + §0.2 撤退方向判定，**不**因仓位低而强卖进攻方向持仓）
- 个体持仓上限（心法 §五：单只 ≤ 30%、同方向 ≤ 50%）**同时**生效
- 整轮流程：先读 `final_analysis.market_overview`（必读）→ 再读 `position_limit` → 用 `position_limit / 100` 作为总仓位上限
- **禁止**：不复述 `position_reasoning` 区间 / 上下沿 / 微调；不基于 `sentiment_label` / `confidence` 调档；不基于"主锚 / 权重 / 核心信号"调档

**开仓触发（两指标联合判定）**

**唯一判定**：是否开新仓由 `sentiment_label` + `sentiment_capital_alignment` 两者联合判定。

> 与心法 §三 仓位哲学的关系：心法四象限（短线差大盘差→禁止）与本节 `alignment=撤退→禁` 是**同一判定的两种表述** —— `sentiment_capital_alignment` 是盘面分析把短线温度与大盘温度综合后的落地信号（共振 / 警惕 / 撤退）。心法哲学是底层依据，本节是可执行判定，不另立第四条件。

**禁开仓条件（两者同时满足）**：

| 字段 | 条件 |
|------|------|
| `sentiment_analysis.sentiment_label` | ∈ {`偏保守`, `保守`} |
| `final_analysis.sentiment_capital_alignment` | == `撤退` |

- 两者同时满足 → **停止开新仓**（禁开仓只作用于"新仓"，**不等于**清掉已有持仓；已有持仓去留归 §三 止盈止损 + §0.2 撤退方向判定）
- 任一不满足 → 仍可按 `position_limit` 执行

> 短线交易每天都会有新题材和持续发酵的方向，情绪温度回答"当前是否适合开新仓、能上多少仓"。具体做哪个题材、位置高低是否接盘，参照盘面分析的资金面（`capital_analysis`）/ 热点轮动（`hot_rotation_analysis`）字段 + 选股买点纪律各自把关，**不**用一个全局风险等级去一概而论地否定开仓。

**判定原则**：
- 只用 `sentiment_label` + `sentiment_capital_alignment` 两指标，不自创"主升 / 回暖 / 混沌 / 退潮"等标签
- `sentiment_label` 单独**不**禁开仓 —— 只有同时叠加 `alignment=撤退` 才禁
- 大盘温度（`temperature_score` / `temperature_label`）**不**作开仓过滤器

### 0.2 活跃方向判定（方向有参考价值，个股名单只是样本，不当门槛）

**活跃方向** = 近期反复活跃、持续发酵的热点题材。来源（取并集，任一即可）：
- `final_analysis.attack_directions[].direction`（盘面分析二次筛选后的进攻方向）
- 顶层 `data["attack_directions"]`（次要参考）
- `hot_rotation_analysis.attack_sectors[].sector`（热点轮动识别的进攻方向）

> **方向 vs 个股名单的辨证（核心认知）**：
> - **方向**（`direction` 字段，如"光通信 / 光纤"）是盘面分析综合资金流、轮动、情绪后给出的**方向性参考**，有决策价值 —— 它帮助柚子 AI 聚焦近期资金主攻的板块。
> - **个股名单**（`direction` 下的 `stocks[].code`）只是盘面分析论证该方向时引用的**样本个股**，受限于喂给它的数据源（成交额榜 / 资金流榜等），**有偏且不完整**。趋势股池里完全可能有形态更优、买点更好、但名单没列出来的标的。
> - **结论**：方向用于聚焦视野与同方向持仓判定；**个股名单不直接进选股池、不作为个股取舍的裁判、不作为仓位分档依据**（§一 选股筛选）。买不买某只股，由趋势形态 + 买点 + 纪律**自主分析**决定。

活跃方向的作用范围（**只做这两件事，不做其它**）：
1. 聚焦选股视野 —— 优先在该方向覆盖的板块里找买点
2. 同方向持仓判定 —— 用于心法 §五（同方向 ≤ 50% 红线）

**不**作选股硬门槛，**不**要求"三角验证同时满足才算活跃方向明确"，**其下的 `stocks[].code` 名单更不是门槛**。

**撤退方向识别**（必须保留，红线）：

盘面分析的 `retreat_directions[].stocks[].code` 只是论证撤退方向时引用的**样本个股**，受限于数据源，**有偏且不完整**。名单漏列不代表脱离撤退方向，名单不卡 —— **判定依据是板块归属 + 主营细分**。

- 调 `cli.py market.get_stock_info <code>` 取 `hot_categories`（热点分类列表）+ `up_reason`（涨停原因 / 主营业务描述）
- 持仓股 `code` ∈ 任意 `final_analysis.retreat_directions[].code`，**或**持仓股 `hot_categories` 中**任一分类**与某 `retreat_directions[].direction` 字面匹配或同板块 → 已撤退方向（红线 0%，立即清仓）
- **同板块内细分方向（应用端 vs 制造端）**：当 `retreat_directions[].direction` 与 `attack_directions[].direction` 共享**同一板块名词**（典型：`CPO/光模块` 既可能出现在撤退方向、也可能以"光通信应用端"形式出现在进攻方向），必须用 `up_reason` 关键字二次细分：
  - **撤退方向细分关键字**（"制造端"特征）：`光模块` / `CPO` / `PCB` / `液冷` / `服务器机柜` / `封装` / `芯片制造` / `800G/1.6T 高速光器件` 等
  - **进攻方向细分关键字**（"应用端"特征）：`光纤` / `光缆` / `光棒` / `光器件应用` / `光通信运营` / `算力租赁` / `AI 应用` 等
  - `up_reason` 含撤退方向关键字 → 已撤退方向（红线 0%）；含进攻方向关键字 → **不**视为撤退方向（属活跃方向）
- 名单（`stocks[].code`）只作交叉验证参考，名单漏列不代表脱离撤退方向

**判定流程优先级**（不能省略任一步骤）：

1. **盘面直接列出的撤退个股**（`final_analysis.retreat_directions[].stocks[].code`）→ 直接判定为撤退方向，**优先剔除**。名单漏列不代表脱离撤退方向，但**名单列出的更应优先剔除**
2. **hot_categories 标签驱动**：仅当 `hot_categories` 包含盘面 `retreat_directions[].direction` 字面词汇（如 `大金融/证券/保险/银行` 或 `光通信/CPO/光模块`）时，才进入下一步细分。**不能因为某股 hot 包含多个标签就只看第一个**
3. **up_reason 二次细分**（仅当步骤 2 命中）：用主营描述判定撤退方向细分关键字 vs 进攻方向细分关键字，**主营细分占优**才判定为撤退
4. **未命中任何判定** → 不视为撤退方向（即使盘面 `attack_directions` 包含相似板块名词）

**常见错误**：
- ❌ 一只股 hot 为 `['CPO', '光通信', 'AI硬件']` → "含 CPO" 直接判撤退。**正解**：进入步骤 3，用 `up_reason` 细分主营是制造端（CPO/光模块）还是应用端（光纤/算力租赁）
- ❌ 一只股 hot 为 `['PCB', 'AI硬件', '业绩线']` → "PCB 在光通信撤退关键词里" 直接判撤退。**正解**：PCB 不在盘面 `retreat_directions[].direction` 字面词汇中（除非盘面明确将 PCB 列入撤退），不应自动判撤退

**同方向持仓判定**（心法 §五 仓位红线前提，仅用于 50% 红线，不用于个股取舍）：
- 持仓股 `code` ∈ 任意 `final_analysis.attack_directions[].stocks[].code`，**或**持仓股所属板块与某活跃 `direction` 同板块 → 视为同方向（上限 50%）。名单只是判定辅助，名单漏列不代表不同方向（板块归属为准）
- 持仓股 `code` ∈ 任意 `final_analysis.retreat_directions[].code` → 已撤退方向（红线 0%）

### 0.3 局部持续热点原则（与心法"条件不够就空仓"的辨证）

**盘面 `temperature_label` 偏冷（退潮 / 冰点等）+ 有近期活跃方向**时，仍按 `position_limit` 直接执行，**不**停止开新仓（禁开仓只作用于新仓，不等于清掉已有持仓）。

- 读 `temperature_label` → 知道大盘温度（仅展示，不做判定）
- 读 `final_analysis.attack_directions` / `hot_rotation_analysis.attack_sectors` → 知道近期活跃方向
- 有近期活跃方向 → 直接按 `position_limit` 执行（**不**主动降到 `position_limit` 以下）
- §0.1 两指标同时走弱（保守 / 偏保守 + alignment=撤退）→ 停止开新仓（已有持仓的去留归 §三 止盈止损 + §0.2 撤退方向判定）

### 0.4 撤退、风险、操作（直接采用盘面结论）

| 盘面字段 | 含义 | 策略动作 |
|----------|------|----------|
| `final_analysis.retreat_directions[].code` | 撤退方向关联个股 | 同方向持仓纳入减仓（红线 0%） |
| `final_analysis.risk_warnings[]` | 风险提示 | 出现任一即纳入决策参考（不单独禁开仓） |
| `final_analysis.action_advice` | 操作建议 | **必读** —— 盘中决策参考 |
| `final_analysis.vs_prev_analysis` | 与上次对比 | 方向逆转 → 持仓纳入盘中实时减仓评估（**信号而非动作**；具体减仓 / 清仓 / 继续持有由下一时点盘中判定决定，不预设"明日开盘第一时间 X 动作"，见总纲 §5.1 第 4 条） |
| `sentiment_analysis.key_signals[看空+强]` | 加重风险信号 | 与 `risk_warnings` 一起纳入决策（总纲 §3.2 key_signals 分级） |
| `hot_rotation_analysis.retreat_sectors[].risk_level == "可能蔓延"` | 撤退风险分级 | **只回避该撤退方向**（不买其个股、持仓减仓）；**不封杀进攻方向开仓**，开仓资格仍归 §0.1（总纲 §5.5）（**仅当整段不为 None 时生效**） |
| `hot_rotation_analysis.retreat_sectors[]` | 资金撤退方向 | 同方向持仓纳入减仓候选；新仓回避该方向个股 |

### 0.5 套利资金面验证

套利标的须满足**主力净流入 > 0 且换手率 ≥ 5%**（资金承接 + 活跃度双重确认，心法 §4.3 套利资金面合规红线）。数据取自三池候选集的 `main_amount` / `turn_z`，或 `cli.py market.get_stock_info <code>`。主力净流出标的**不**进套利评估。

### 0.6 加速止盈分时形态判定（分时字段）

加速止盈是**形态信号**，只能从分时序列 + 主力净额识别，**禁止用持仓快照的涨跌幅 / 浮亏% 代替**（给不出形态，见总纲 §3.3 与 §5.1 第 5 条）。对每只持仓按需调 `cli.py market.get_stock_minute_trendline <code>`（分时：`labels` 时间序列 + `y1.data` 涨跌序列 + `y2.data` 主力资金序列，`y2` 末值为当日主力净额）+ `cli.py market.get_stock_info <code>`（板块归属 `hot_categories` / `up_reason`），按下表判四信号：

| 信号 | 判定 |
|------|------|
| 放量滞涨 | 冲高回落 / 不创新高 **且** 量能未萎缩 |
| 长上影线 | 日内最高涨幅 − 收盘涨幅 **≥ 4%** |
| 走势与板块背离 | 所属 `attack_directions` 当日集体走强，个股却冲高回落 / 逆势走弱 |
| 资金背离 | 涨幅 > 0 **但** `y2` 末值 < 0（主力净流出，小单拉大单出）→ 预警，**滞涨转跌**才确认卖出 |

任一信号成立即加速止盈。出现任一异常默认倾向卖出、需明确理由才留（心法 §4.5 六条心得"优先保住利润"）。

## 一、选股筛选

### 1.1 候选集（唯一来源，三池全拉合并）

**唯一来源**：`cli.py market.get_strategy_trend_stocks <strategy_id> limit=30`，返回 `data` 为 `list[dict]`，每项含 `stock_code/stock_name/zdf/price/turn_z/amount/main_amount/intraday`。

**取数方式（三池全拉合并分析，不存在"主池优先、其余兜底"）**：

| 池 | 调用 | 覆盖范围 |
|----|------|----------|
| 创业板强趋势 | `get_strategy_trend_stocks 6` | 10 日内涨幅 > 7%，创业板，非 ST |
| 创业+科创大趋势 | `get_strategy_trend_stocks 10` | 15 日内 > 9%，涨幅 > 3%，覆盖创业+科创板 |
| 沪深主板强趋势 | `get_strategy_trend_stocks 7` | 沪深主板强趋势 |

> 三个池子覆盖的板块结构不同 —— 创业板偏成长 / 科技、科创板偏硬科技、主板偏周期 / 大市值 / 金融。**活跃方向的核心标的可能落在任一池**，只看其中一池就停止，等于自废视野。**正确做法：三池全部拉下来，按 `stock_code` 合并去重，形成一个统一候选集，再做买点分析**。

- 三池**全部拉取**，按 `stock_code` 合并去重为统一候选集（典型 80~100 只），**不存在"主池不足才补"的降级逻辑** —— 主池(6)充足也必须继续拉 10 和 7
- 合并后在统一候选集上做：①剔除撤退方向个股 ②剔除已涨停（不打板）③按 §二 判定买点 ④结合 §0.2 活跃方向做**仓位权重参考**
- 合并候选集经①②③过滤后无可买标的 → 输出"无候选标的"并按 §0.1 两指标联动 + §三 止损止盈继续执行（**不**跳过持仓检查，**不**静默空仓）

**约束**：
- 选股池**只来自上述接口**。**不**从盘面 `attack_directions` / `attack_sectors.key_stocks` / `get_hot_spot_components` / 连板梯队等任何来源选股
- 候选池**不**做活跃方向归属验证。趋势股池票只要同时满足①不在 `retreat_directions`（已撤退方向，红线 0%）②不在五类股（接口入库隐含过滤，盘中异常用 `get_stock_info` 兜底）③出现 §二 买点，即可买。**不**要求标的必须落在盘面活跃方向名单内
- **个股取舍由趋势形态 + 买点 + 纪律自主判定，不查盘面活跃方向名单**。盘面活跃方向（方向名）有参考价值，但其下个股名单只是盘面论证方向用的**样本**，**有偏且不完整** —— 形态合格、有买点但不在名单内的标的，同样可以买
- **单只仓位统一上限 ≤ 30%**（心法 §五 仓位红线），具体多少由买点质量、距止损位空间、`risk_warnings` 提示的追高风险**自行研判**：买点位置理想、空间充足给满 30%；尾盘追高、位置偏高则主动收敛，但收敛依据是**位置与风险**，不是**名单归属**
- 龙头是心法层面的优先目标（辨识度最高、带动性最强），龙头无买点时退而求其次做活跃方向内的中军 / 活跃标的
- 取得的候选股已通过五类股过滤（接口入库隐含完成），直接按 §二 判定买点

**口径说明**：`strategy_id=10` 对应 `app/compute/strategies/strategy_jrqs_dqs.py`（"大趋势"：创业板或科创板、涨幅 > 3%、15 日内 > 9%）。

### 1.2 龙头套利专项（活跃方向内中军 / 活跃股补涨）

> **触发场景**：龙头（板块辨识度最高的标的）已涨停 / 当日涨幅 > 7% 高位 / 错过买点 → 资金溢出到同方向同板块中军 / 活跃股的跟随补涨机会。
>
> **边界红线（方向 / 资金面 / 位置 / 买点类型）见心法 §4.3**。本节是执行层细化 —— 明确四要素联合判定 + 优先级排序，避免把"任何同方向上涨的股"都当成套利。

**四要素联合判定（缺一不可，红线见心法 §4.3）**：

| 要素 | 条件要点 | 验证方式（策略执行层） |
|------|------|----------|
| **① 方向归属** | 同方向同板块 / 主营细分重合 | `get_stock_info` 取 `hot_categories` + `up_reason` 与龙头方向对比 |
| **② 资金面硬条件** | 主力净流入 > 0 且 换手 ≥ 5% | 候选集 `main_amount` > 0 且 `get_stock_info` 取 `turn_z` ≥ 5（§0.5） |
| **③ 位置红线** | 当日涨幅 ≤ 7% | 候选股 `zdf` 直接判定 |
| **④ 买点类型** | 分时承接 / 突破超预期买点 | 分时回踩均价线不破后快速拉起 / 放量突破前高（见 §二） |

**优先级排序（与其它候选冲突时）**：

```
低吸区龙头股（当日涨幅 -3% ~ +3% + 龙头）
  > 强势可承接区套利（当日涨幅 3% ~ 7% + 同方向中军 + 资金净流入 + 分时承接 / 突破买点）
  > 跟风股（不评估）
```

**禁止的套利**：
- ❌ 跨方向套利（"AI 算力龙头涨停 → 买半导体方向"等）
- ❌ 资金面流出套利（"主力 -X 亿但我看好"）
- ❌ 当日涨幅已 > 7% 的位置破红线套利
- ❌ 跟风脉冲套利（"龙头封板带动同板块脉冲 3-5% 但无独立逻辑"）

**判定流程**：
1. **识别龙头**：板块内辨识度最高的标的（不依赖盘面标记）= 当日涨幅最高 + 主力流入最强 + 市值 / 流通市值前列
2. **龙头是否有买点**：当日涨幅 ≤ 7% + 满足 §二 买点之一 → **直接买龙头**（优先级最高）
3. **龙头无买点**（涨停 / 涨幅 > 7% / 错过买点）→ 进入套利评估池
4. **套利评估池筛选**（四要素联合）：①同方向 ②主力净流入 + 换手 ≥ 5% ③涨幅 ≤ 7% ④分时承接 / 突破买点
5. **未命中任一要素** → 输出"无可套利标的"

**接口隐含的过滤项**（候选集入库时已保证，**不重复校验**）：

| 过滤项 | 入库筛选标准 |
|--------|--------------|
| 趋势 | 均线多头排列（价 > MA5 > MA10 > MA20 > MA30）+ 间距发散 + MA30 上移 |
| 股性 | 近 120 个交易日内至少 1 次涨停 |
| 形态 | 15 日内阳线多于阴线 + 近 10 日涨幅 > 7% |
| 基本面 | 非 ST、无可转债、自由流通市值 > 30 亿 |

> 候选集由定时任务刷新，存在滞后。盘中若发现候选股临盘异常（突发破位、停牌），用 `get_stock_info` 单票兜底，不维护静态过滤表。

### 1.3 已涨停判定（区分板块）

候选股代码前缀决定涨停板阈值：
- 主板（`60` / `00` 开头）：`zdf ≥ 9.9` → 已涨停（不打板）
- 创业板（`30` 开头）/ 科创板（`68` 开头）：`zdf ≥ 19.9` → 已涨停（不打板）
- 阈值不能统一为 `zdf ≥ 9.9` —— 会误杀创业板 / 科创板 10-19% 的强势未封板股

## 二、买点规则（不打板、不接力、只做超预期）

**统一原则**：

四个买点都共享"**超预期**"内核 —— 标的在关键位置出现超出市场预期的表现。**超预期不等于"只在竞价发生"**，四个买点 1/2/3/4 在盘中**任何时点都可能成立**（买点 1 仅在竞价阶段有效；买点 2/3/4 盘中任何时点都成立）：

- **买点 1（竞价超预期）**：仅在 9:15-9:25 竞价阶段观察；早盘 9:30 后失效
- **买点 2（分时承接超预期）**：盘中任何时点都成立；**可用于龙头套利场景**（同方向中军分时回踩均价线不破后快速拉起，伴随成交量持续放大）
- **买点 3（突破超预期）**：盘中任何时点都成立；**可用于龙头套利场景**（同方向中军放量突破前期高点 / 整理平台，成交量 > 近 5 日均量 1.5 倍）
- **买点 4（低吸）**：见 §2.2，盘中任何时点都成立

| 优先级 | 买点类型 | 条件 | 适用场景 |
|--------|----------|------|----------|
| 1 | 竞价超预期 | 昨日分歧/调整，今日竞价放量高开（>2%），竞价量 > 昨日全天 5%，**且未涨停** | 龙头低吸 / 竞价超预期 |
| 2 | 分时承接超预期 | 开盘后分时回踩不破均价线或前日收盘价，快速拉起，成交量持续放大 | **龙头套利**（同方向中军分时承接）/ 低吸区龙头股 |
| 3 | 突破超预期 | 放量突破前期高点/整理平台，成交量 > 近 5 日均量 1.5 倍 | **龙头套利**（同方向中军突破前高）/ 低吸区龙头股 |
| 4 | 低吸（见 §2.2） | 攻击方向内强势股早盘/午后回踩 -2%~-3%，未破 MA5/MA20，多头排列，板块仍强 | 低吸区龙头股 / 套利标的回踩 |

**纪律**（来自心法，不可妥协）：
- 已经涨停 → 不追
- 已经连板 → 不追
- **当日涨幅 > 7%（接近涨停或连板）→ 一律视为高位，不追**（不区分板块；心法 §4.3 套利位置红线）
- 错过买点 → 放弃，等下一个
- 买入后立即设止损；不设止损 = 不买入
- **套利标的必须满足 §1.2 四要素联合判定**（方向归属 + 资金面 + 位置 + 买点 2/3）

### 2.2 低吸买点

**核心定位**：低吸是"**攻击方向内部**的强势股回踩机会"，不是"冷门股抄底"，也不是"高位股追涨"。

**判定条件（正向买入理由 + 前置否决过滤两组）**：

> 第 5、6 条为前置否决过滤：任一成立即否决买入，但**不是买入理由，禁止写入买入 reason**；买入 reason 只写第 1~4 条（§7.1）。

**① 正向买入理由（全部满足）**：
1. **属于近期攻击方向**（命中 `final_analysis.attack_directions[].direction` ∪ `hot_rotation_analysis.attack_sectors[].sector`）
2. **形态多头排列**：close > MA5 > MA10 > MA20（任一破位即不符合）
3. **回踩幅度合理**：当前 `zdf` 在 **-3% ~ +3%** 区间（强势股的小阴/小阳整理），**不**接 -3% 以下的飞刀
4. **板块仍强**：当日盘面进攻方向强度未变弱（`vs_prev_analysis` 未报告该方向降级）

**② 前置否决过滤（任一成立即否决买入）**：
5. **分时未现卖点（硬前置，不可跳过）**：买入前**必须**对候选股调 `get_stock_minute_trendline` 采分时（涨跌 `y1` + 主力 `y2`），按 §0.6 判四信号（放量滞涨 / 长上影 / 走势与板块背离 / 资金背离），任一成立即**否决买入** —— 与持仓监控（教训 A.7）同等强度，禁用 `zdf` 快照 / K线代替分时判定（教训 A.8）
6. **资金-涨幅不背离**：主力大额净流入（量级显著，如 > +1 亿）但 `zdf` 压在低位（资金量级与涨幅明显不匹配）→ "钱进价不涨"的派发嫌疑，**优先怀疑派发而非低吸**，需第 5 条分时确认是真回踩还是冲高回落（心法 §4.5）

**时间窗口**：
- **早盘冲高回落**：10:00-11:30 是最佳窗口（强势股早盘冲高后回踩均价线 / 前日收盘；起点避开 9:30-10:00 高风险时段，此时方向未明、洗盘剧烈，`zdf` 易跌破 7%）
- **午后分歧低吸**：13:00-14:00 资金回流时回踩买点
- **尾盘 14:30 之后**：非低吸窗口（尾盘再追入属追涨，违反"不追高"）

**纪律**：
- ❌ "低吸" ≠ 抄底下跌趋势（心法 §4.1：20 日线下方不接飞刀）
- ❌ "低吸" ≠ 在冷门题材 / 无主流资金关注的股上买入
- ❌ 低吸买点出现时不调高单只仓位上限（仍按心法 §五 ≤ 30%）

## 三、止损止盈（个股级硬阈值）

### 3.1 主动评估原则与阈值表

**不止被动等触发，所有止盈维度每轮盯盘都必须主动评估** —— 不等"破止损位 / 破 20 日线"再动作。**不及预期提前获利了结**是纪律红线，不是事后止损。**优先保住利润，非强势股少格局** —— 出现放量滞涨 / 走势与板块背离即倾向落袋，不死扛等大涨。

> **加速止盈是形态信号，只能从分时序列识别（见 §0.6），禁止用收盘快照价 / 浮亏% 代替**。

| 评估时机 | 评估维度 |
|---------|---------|
| 每轮盯盘 | ① 加速止盈信号（放量滞涨 / 长上影线 / 走势与板块背离，见 §0.6）——**不等触顶才看** |
| 每日收盘 | ② 时间止损（买入后第 3 日评估是否出现预期上涨；冲高回落不算，见 §0.6） |
| 浮盈时 | ③ 目标止盈（+10%）、④ 利润保护（从最高点回撤 -5%）、⑤ 移动止损（+50% 后回撤 -10%） |

| 类型 | 触发 | 执行 |
|------|------|------|
| 固定止损 | 买入价下跌 -5% | 立即卖出 |
| 技术止损 | 跌破买入日最低价或关键支撑（20 日线、前低） | 立即卖出 |
| 时间止损 | 买入后 3 日内未出现预期上涨（**冲高回落不算预期上涨**，见 §0.6） | 评估后离场 |
| 趋势止损 | 股价跌破 20 日线 | 立即卖出 |
| 目标止盈 | 浮盈 +10% | 减仓一半 |
| 加速止盈 | 放量滞涨 / 长上影线 / 走势与板块背离（见 §0.6） | 清仓或大幅减仓 |
| 利润保护 | 浮盈从最高点回撤 -5% | 清仓保护利润 |
| 移动止损 | 单只盈利 > +50% 后回撤 -10% | 清仓 |

> **开盘极端行情止损确认窗口**：开盘大低开 / 秒杀瞬间击穿止损位，**不机械秒割**，给约 15 分钟（9:30-9:45）确认 —— **缩量急跌快速收回 = 恐慌洗盘，不止损；放量跌破买入日最低价 / 20 日线不回头 = 趋势破位，确认止损**。触跌停、放量破 20 日线 → 立即止损不等。（固定止损在开盘极端行情下的执行细则，不废除止损纪律。）

### 3.2 加速止盈信号识别（每轮持仓必判）

> 对每只持仓取 `get_stock_minute_trendline`（涨跌 `y1` + 主力 `y2`）+ `get_stock_info`（板块归属），按 §0.6 四信号表判定，任一信号成立即加速止盈 —— **禁用快照价 / 浮亏% 判定**（给不出形态与资金，教训 A.7）。

- **时间止损**：冲高回落（最高 − 收盘 ≥ 4%）不算"预期上涨已现"，触发离场（防"收盘红盘"的伪正向）
- 出现任一异常默认倾向卖出、需明确理由才留（心法 §4.5）

| 趋势股池 | 三池合并结构：**创业板强趋势(6) + 创业/科创大趋势(10) + 沪深主板强趋势(7)**，三池全部拉取按 `stock_code` 合并去重为统一候选集（§1.1，不存在"主池优先、其余兜底"） |
| 活跃方向 | 消费方式见总纲 §3.2 / §5.1 / §5.5。**方向名**用作视野聚焦；名单**不**作个股取舍裁判 / **不**作仓位分档依据（§1.1、教训 A.5） |
| 已撤退方向 | 板块归属为准 + 细分配套关键字判定（§0.2）；判定命中 → **红线 0%**（心法 §五 仓位红线） |
| 套利标的 | **四要素联合判定**（§1.2）：①与龙头同方向同板块 ②主力净流入 + 换手 ≥ 5% ③当日涨幅 ≤ 7% ④分时承接 / 突破买点 |
| 套利 vs 低吸区龙头股 | **优先级**：低吸区龙头股（涨幅 −3% ~ +3% + 龙头） > 强势可承接区套利（涨幅 3% ~ +7% + 同方向中军 + 资金净流入 + 分时承接 / 突破买点） > 跟风股（不评估）。详见 §1.2 |
| 高位 | 本账号阈值：**当日涨幅 > 7%**，不按板块区分；> 7% 一律不追（心法 §4.3 套利位置红线、§二 买点纪律） |
| 低吸买点 | 时间窗口：早盘 **10:00-11:30** / 午后 **13:00-14:00**；尾盘 14:30 后非低吸窗口（§2.2） |
| 已涨停 | **按板块区分阈值**：主板 ≥ 9.9% / 创业板 / 科创板 ≥ 19.9%（§1.3）。术语本身的板块阈值见总纲 §六 |
| position_limit | 盘面分析给出的当日仓位上限百分比（0-100），**直接采用**（§0.1），不自行调档 |

## 七、输出规范

每轮盯盘输出由「盯盘总结标记」起头，后接观点总结 + 一行摘要。**注意：盯盘的"开始/结束"边界由调度日志自动打印，agent 不再重复输出"盯盘开始/盯盘结束"，只用「盯盘总结」标记自己的分析输出**：

```
========== 盯盘总结 HH:MM:SS ==========
（盯盘观点总结）
（一行摘要行）
```

- **盯盘总结标记**：观点总结的标题行，标记本轮分析输出开始
- **盯盘观点总结**：结构化分析结论（见下方板块结构）
- **一行摘要行**：`[柚子 AI] HH:MM | ...` 摘要（见下方格式）

> **标记格式固定**：`========== 盯盘总结 HH:MM:SS ==========`（等号 10 个，时间精确到秒，取本机当前时刻）。**不要输出"盯盘开始/盯盘结束"** —— 轮次边界由调度日志负责，agent 重复输出会造成两份重复标记。

> **核心要求**：观点总结**按板块给结论、过程数据作支撑**，禁止把 §五 checklist 的逐项检查过程（"§0.1 两指标联合 / 趋势股池+过滤 / 方向归属核验…"这类流水账检查项表格）直接当输出。读者要一眼看到「观点」，而不是在一堆检查项里自己拼结论。

### 7.1 观点总结板块结构

| 板块 | 内容（先结论，后可验证支撑） | 空值兜底 |
|------|------------------------------|----------|
| **市场判断** | 情绪（`sentiment_label`）+ alignment（`sentiment_capital_alignment`）→ §0.1 开仓资格结论；仓位上限（`position_limit`）；活跃方向；一句话市场核心特征 | —— |
| **持仓个股分析** | 逐只：代码/名称、`zdf`、浮盈、§三 止盈止损评估结论、本轮动作（持有/减仓/清仓）。**必须带分时形态结论**（§0.6 四信号：放量滞涨/长上影/板块背离/资金背离），禁只用快照价判定 | 写"当前空仓" |
| **候选个股分析** | 三池合并去重总数 → 过滤项（已涨停/高位 `zdf`>7%/重点监控禁买名单/撤退方向）→ 进入核验数；逐只取舍结论（代码/名称、`zdf`、命中买点类型或"无买点+原因"、方向归属判定结果）。盘面 `action_advice` 推荐但被纪律否决的龙头在此说明原因（如已涨停不打板、不在三池） | 写"三池合并 N 只，过滤后无符合买点标的" |
| **综合决策** | 本轮最终动作 + 一句话核心原因（可验证事实，非主观判断） | —— |

纪律（观点总结同样受总纲 §5.6「不允许的写法」约束）：

- 情绪/仓位/活跃字段直接采用盘面返回值，**不**自创标签/调档数字，**不**复述 `position_reasoning` 区间
- 候选取舍只报客观事实（`zdf`、买点类型、方向归属判定结果），**不**写"我判断不匹配/我认为不行"等主观话术
- **不**预设未来时点动作；可写"明日需重新评估的检查项"清单（不绑定动作），禁止把 `vs_prev` 信号机械外推为下一时点具体动作
- **不**自拾炸板率/大面家数/破位率/连板高度等指标作为论据
- **买入 reason 与卖出 reason 不得混写（写法红线）**：
  - 买入 reason 只写正向买入逻辑：方向归属 + 买点类型 + 形态/量能事实（§2.2 ①组、§二 买点）
  - 卖出 reason 只写卖出逻辑：止损/止盈类型（§三）、加速止盈四信号（§0.6）、撤退方向（§0.2）
  - 禁止把"分时未现卖点""未触发止损""未现止盈信号""T+1 不可卖"等否决/持有侧逻辑写入买入 reason

观点总结示例（空仓、无买点场景）：

```
【市场判断】情绪偏保守 + 警惕（非撤退）→ 不禁开仓，仓位上限 30%；活跃方向存储芯片/先进封装；大盘系统性风险释放中（锂电崩盘）
【持仓分析】当前空仓
【候选分析】三池合并 19 只去重 → 过滤已涨停 1 / 高位(zdf>7%)5 / 重点监控禁买 2 → 11 只核验；昆仑万维(AI应用,方向不符)/巨人网络(游戏,方向不符)；格科微(芯片✓) zdf+2.88% 但芯片方向降级(§2.2 条件4不满足)；候选股均冲高回落无买点 2/3。盘面推荐华天科技/浪潮信息均 zdf≈10% 已涨停不打板，兆易创新/寒武纪不在三池
【综合决策】空仓不操作（候选股均冲高回落无买点 + 芯片方向降级）
```

### 7.2 一行摘要行（观点总结之后输出）

格式（与总纲 §八 一致）：

```
[柚子 AI] HH:MM | 阶段 | 情绪:激进/偏激进/中性/偏保守/保守 | 置信度:X | 活跃:XX | 仓位:NN% | 操作:XX | 持仓:NN只
```

- 情绪字段直接采用盘面 `sentiment_label` 值，**不**自创"主升/回暖/退潮"等标签
- 仓位字段直接采用盘面 `position_limit` 值，**不**自创"5成""3成"等调档数字
- 活跃字段 = `final_analysis.attack_directions[0].direction if final_analysis.attack_directions else "该字段为空"`（仅展示近期活跃方向，不复述 `reason` / `evidence`；缺位时按总纲 §5.2 字段缺位判定）
- 操作字段 = 实际动作；"无操作"也要输出
- **不**输出 `position_reasoning` 的区间/上下沿/微调表述

### 7.4 复盘输出规范（标记 + 板块 + 摘要）

盘后复盘输出与盯盘对称：由「复盘总结标记」起头，后接 4 个结构化板块 + 一行摘要。**复盘同样用「复盘总结」标记自己的产出，不输出"复盘开始/复盘结束"边界**（边界由调度日志负责）：

```
========== 复盘总结 YYYY-MM-DD ==========

【当日盈亏】（今日盈亏 + 累计 + 交易笔数 + 持仓明细）
【情绪演变】（情绪 / 温度 / 主力净额当日轨迹）
【操作反思】（合规检查 + 纪律执行 + 经验）
【次日计划】（次日倾向 + 检查项清单 + 活跃方向）

[柚子 AI] 复盘 | 盈亏:... | 反思:... | 次日活跃:...
```

- **复盘总结标记**：`========== 复盘总结 YYYY-MM-DD ==========`（等号 10 个，日期取当日交易日）

复盘板块结构（先结论、后可验证支撑，同样受总纲 §5.6 纪律约束）：

| 板块 | 内容 | 空值兜底 |
|------|------|----------|
| **当日盈亏** | 今日盈亏金额（+比例）、累计盈亏；交易笔数；持仓明细（每只代码/名称/浮盈%） | 写"今日无交易" |
| **情绪演变** | 当日情绪/温度/主力净额的轨迹（起→终），点明情绪转折；**只引用盘面字段**（`temperature` / `sentiment` / `main_amount`），**不**自拾炸板率/大面家数等额外指标（总纲 §5.1 第 2 条） | —— |
| **操作反思** | 合规性检查（总仓位 vs 盘面 `position_limit`、单只 ≤30%、同方向 ≤50%、已撤退方向=0）；纪律执行情况；可复用经验/教训（对应 `daily-summary` 的 `reflection`） | 写"今日无操作，无反思项" |
| **次日计划** | 次日仓位倾向、**需重新评估的检查项清单**（不绑定动作，呼应教训 A.6「信号≠动作」）、次日活跃方向（对应 `daily-summary` 的 `next_day_plan`） | —— |

```
========== 复盘总结 2026-07-09 ==========

[柚子 AI] 复盘 | 盈亏:+0.7% | 反思:加仓合规 | 次日活跃:存储芯片+先进封装+CPO
```

> 复盘摘要行格式：`[柚子 AI] 复盘 | 盈亏:+X.X% | 反思:<一句话> | 次日活跃:<方向>`；盈亏取实盘结算值、活跃取盘面方向值，**不**自创。

### 7.5 总结上报（输出后必做）

每轮总结产出后，**必须**调 `report.submit_summary` 把当轮总结文本同步到后台（写入 `mock_log` 表），供事后回溯 agent 的判断与决策。**每轮盯盘都要上报（含"无操作"轮）**，复盘产出后上报一次。

| 阶段 | 调用 | `content` 取值 |
|------|------|----------------|
| 盯盘（每轮输出后） | `cli.py report.submit_summary watch "<文本>"` | 当轮「盯盘总结标记」+ 观点总结 + 摘要行的完整文本（**含标记**） |

**示例**（content 含标记与换行，务必用单引号整体包裹，避免 shell 对换行/引号的转义破坏）：

> - `log_type` 仅 `watch`（盯盘）/ `review`（复盘），与阶段一一对应
> - **失败隔离（硬约束）**：上报是本轮流程的**最后一步且完全旁路**。`report` 已保证上报请求 10s 超时（`_post` timeout）并吞掉**一切**异常返回错误信封（`code ≠ 200`）—— 上报在技术上不可能让本轮卡死、抛异常或中断

## 附录 A：盯盘教训（经验教训）

> 本节是历史盯盘中沉淀的**经验教训**，是总纲通用纪律（§5.1 五条硬纪律）与心法纪律的**具象化案例**，供 agent 避免重蹈覆辙。后续是否增删由用户指示，agent 不得自动追加新案例。

### A.1 教训一：误用"主锚"概念 → 仓位完全采用盘面分析，不自创名词

- **错误表现**：盯盘中 agent 写"大盘温度是主锚"作为不开仓的论据
- **根因**：禁止自创判定名词，但 agent 仍在论据中自创概念，违反输出纪律
- **正解**：仓位完全采用盘面 `final_analysis.position_limit`，**不**自创调档。情绪判定只引用盘面返回的 `sentiment_label` / `sentiment_capital_alignment` 两指标（§0.1）

### A.2 教训二：自拾额外指标 → 情绪判定只用盘面两个温度

- **错误表现**：盯盘记录中 agent 列出炸板率 33.3%、大面 57 家、昨日涨停破位 23.5% 等指标作为"情绪背离"的论据
- **根因**：心法明令"不自行统计炸板率、跌停数、连板高度"等指标，但 agent 在压力下仍自拾这些字段来"证明情绪差"
- **正解**：情绪判定**只**用盘面的两个温度 + alignment（总纲 §5.1 第 2 条）：
  - 大盘情绪温度：`temperature_score` + `temperature_label`（只用于 `position_limit`）
  - 短线情绪温度：`sentiment_label` + `sentiment_capital_alignment` 决定是否开新仓（§0.1 两指标）
  - 炸板率、大面家数、破位率、连板高度、涨跌停家数等**一律不准自拾**作为情绪论据

### A.3 教训三：复述 `position_reasoning` 区间 → 诱发自创调档

- **错误表现**：盯盘输出复述"盘面给出 65-80% 区间，本日取中下沿 65%"——**复述本身就在做"取档"动作**
- **根因**：把 `position_reasoning` 当作"可读"字段，但盘面分析在 reasoning 中写"区间+微调"是它的内部推理过程，agent 复述就会养成"区间"概念
- **正解**：`position_reasoning` 仅供日志/解释，**禁止**复述区间/上下沿/微上修/微下修等表述。盯盘输出**只**引用最终数字 `position_limit`（总纲 §5.6、§7）

### A.4 教训四：候选池为空就什么都不做

- **错误表现**：盯盘中 `get_strategy_trend_stocks(6)` 创业板强趋势过滤后无可买标的 → agent 直接输出"无候选"并**中断检查链**，跳过持仓止损止盈检查、跳过输出摘要
- **根因**：把"取候选池"误读为"全部操作"，忽略了下游的持仓检查
- **正解**：选股接口是**三池全拉合并**（6 + 10 + 7），三池全部不足才"无候选"；但即使"无候选"也要**继续执行**持仓止损止盈检查（§三）+ 输出摘要（§七）+ 盘面 `action_advice` 解读（§0.4）。**取候选池只是检查链的一个节点，不是全部**（§1.1、§五 checklist）

### A.5 教训五：把盘面个股名单当白名单 + 只看主池就停

- **错误表现**：盯盘中 agent 先只拉 `get_strategy_trend_stocks(6)` 创业板主池就停止分析（用户指出后才补拉 10/7）；并且把"是否落在 `final_analysis.attack_directions[].stocks[].code` 名单内"作为个股取舍与仓位分档依据（名单内单只≤30%、名单外≤20%）
- **根因（双重错误）**：
  1. **只看主池**：误读 §1.1「三池全拉」为"主池够数就停"，忽略了三个池覆盖的板块结构不同（创业板偏成长、主板偏周期/大市值），活跃方向核心标的可能落在任一池。主池充足≠视野完整
  2. **名单当门槛**：把盘面的 `attack_directions[].stocks` 当成"白名单"，忘了它只是盘面论证方向时引用的**样本**，受限于数据源，**有偏且不完整**。趋势股池里形态更优、买点更好但不在名单内的标的，被这个隐门槛挡掉了
- **正解**：
  1. **三池全拉合并**：`get_strategy_trend_stocks(6)` + `(10)` + `(7)` **全部拉取**，按 `stock_code` 合并去重为统一候选集，再做买点分析（§1.1）
  2. **方向有用、名单不卡**：盘面的 `direction`（方向）有参考价值（聚焦视野），但其下 `stocks[].code` **不进选股池、不作个股取舍裁判、不作仓位分档依据**。买不买由趋势形态 + 买点 + 纪律**自主判定**（§0.2、§1.1，总纲 §5.1 第 3 条）
  3. **单只仓位统一 ≤30%**，具体多少由买点位置与追高风险自行研判，**不**按名单内/外分档（§1.1）

### A.6 教训六：把撤退名单当白名单 + 方向逆转信号外推为机械动作

- **错误表现**：
  1. 盯盘 600487 亨通光电时，检查 `retreat_directions[].stocks[].code` 名单里没有 600487，输出"持仓不在撤退方向"——实际亨通光电主营光模块/CPO，属撤退方向 `CPO/光模块/AI硬件制造端` 板块归属范围
  2. 观察到 `vs_prev_analysis` 报告"CPO/光模块制造端撤退力度加大"，机械外推为"明日开盘第一时间减仓"——把盘中信号当作下一时点动作的预设承诺
- **根因**：
  1. **名单当白名单的延伸**：A.5 警告了"攻击方向名单当门槛"，本条扩展到"撤退方向名单当白名单"——任何盘面名单都是论证样本，不能作为个股归属的最终裁判
  2. **信号外推成动作**：把"观察到的状态变化"自动映射成"未来必须执行某动作"，忽略了"下一时点盘中要重新跑判定"的基本原则
- **正解**：
  1. **板块归属判定**：`get_stock_info(stock_code).hot_categories` + 主营业务描述 vs `retreat_directions[].direction` 字面匹配/同板块 → 视为该撤退方向（红线 0%）；名单只作交叉验证（§0.2 撤退方向识别、心法 §五）
  2. **信号 ≠ 动作**：`vs_prev_analysis` 报告方向变化是**待评估的盘中信号**，下一时点的具体动作（减仓 / 清仓 / 继续持有）由下一时点的盘中判定决定（§0.4 撤退/风险/操作表、总纲 §5.1 第 4 条）
  3. **输出只描述当下**：盯盘一行摘要只陈述当前事实 + 明日需重新评估的检查项清单，**禁止**承诺式动作预设（§7、总纲 §八）

### A.7 教训七：持仓只用快照价判定 → 加速止盈是形态信号，必须采集分时

- **错误表现**：盯盘 300083 创世纪时，只用 `get_positions`+`get_batch_stock_zdf` 的快照价 / 浮亏%（`zdf` +1.48%、浮亏 -1.87%）判定"未触发止损、时间止损通过、继续持有"，**未采集分时**。实际该股当日早盘冲高 +7.05% 后全天阴跌回落至 +1.26%，形成长上影 + 放量滞涨 + 与机器人板块集体走强（+82 亿、涨停 34 只）背离，明确构成 §0.6 加速止盈信号，被用户指出后才纠错卖出
- **根因（双重）**：
  1. **数据采集结构性失衡**：给候选股拉了分时认真看买点，给持仓股却只取快照——"重买入决策、轻持仓监控"。而加速止盈（长上影 / 放量滞涨 / 板块背离）只能从分时序列识别，快照价给不出形态
  2. **"伪正向"误判**：用"今日 `zdf` > 0 的收盘红盘"论证"预期上涨已现、时间止损通过"，掩盖了过程上的冲高派发——冲高回落在收盘 `zdf` 上看起来也是"涨"
- **正解**：
  1. **持仓分时与主力资金强制采集**：每轮持仓复核对每只持仓调 `get_stock_minute_trendline`（涨跌 `y1` + 主力 `y2`）+ `get_stock_info`（板块归属），按 §0.6 判四信号（含资金背离），禁止只用快照价判加速止盈（总纲 §3.3、§5.1 第 5 条）
  2. **上涨质量判别**：时间止损的"预期上涨已现"必须是稳健上行 / 高位横盘；冲高回落（日内最高 − 收盘 ≥ 4%）不算，触发离场
  3. **板块背离即异常**：所属 `attack_directions` 当日集体走强、个股却冲高回落 / 逆势走弱 → 走势与板块背离，加速止盈（不依赖主观的"板块地位"判定）
  4. **优先保利润少格局**：非强势股出现异常默认倾向卖出，不死扛（心法 §4.5）

### A.8 教训八：买入只用 zdf 快照判定 → 分时未现卖点同样要采分时（A.7 的买入镜像）

- **错误表现**：盯盘 300604 长川科技时，以"主力 +2.24 亿 + 买点 4 低吸区 `zdf` +2.73% + 方向归属命中"买入，**未采分时**。实际该股主力大额进场却涨幅压在低位（钱进价不涨），属放量滞涨 / 派发嫌疑，买入后仅 9 分钟（10:49）即触发长上影 4.51% + 放量滞涨 + 板块背离加速止盈信号，因 T+1 锁定无法卖出
- **根因（双重）**：
  1. **买入与持仓的分时采集不对称**：持仓每轮强制采分时判四信号（教训 A.7 + §五 checklist 持仓分时采集项），买入候选却只查 `get_stock_info` / K线，第 5 条「分时未现卖点」流于形式——同一个放量滞涨形态，持仓认得出、买入认不出。A.7 补了"持仓不采分时"，漏了"买入不采分时"
  2. **主力净流入被当无条件加分项**：把"主力 +2.24 亿"当正向证据，忽略了"大额进场却只涨 2.73%"恰恰是资金-涨幅背离的派发信号；原有规则只有资金背离止盈（价涨资金出，§0.6），缺"资金-涨幅背离"的买入否决（资金进价滞涨）
- **正解**：
  1. **买入分时与主力资金强制采集（对称 A.7）**：买入候选必须调 `get_stock_minute_trendline`（涨跌 `y1` + 主力 `y2`）+ `get_stock_info`（板块归属）按 §0.6 判四信号，任一成立否决买入（§2.2 第 5 条硬前置、§五 checklist 买入候选分时采集项）
  2. **资金-涨幅背离即派发嫌疑**：主力大额净流入但涨幅与资金量级明显不匹配 → 优先怀疑派发而非低吸，分时确认（§2.2 第 6 条）
  3. **放量滞涨买卖对称**：铁律 7「放量滞涨就走」的判断同时适用于买入侧——分时已现放量滞涨 / 长上影的标的不买（心法 §4.5）
  4. **`zdf` 落在区间 ≠ 低吸成立**：买点 4 第 3 条的 -3%~+3% 是必要非充分条件，必须叠加第 5 条分时确认，禁用涨幅快照短路判定
